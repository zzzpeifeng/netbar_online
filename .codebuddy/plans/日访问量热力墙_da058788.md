## 产品概述

在Web「设置」页面新增日访问量热力墙，以GitHub Contributions风格展示网站每日独立IP访问量。深色主题搭配紫色到粉红色渐变，帮助管理员直观了解网站的访问活跃趋势。

## 核心功能

- **访问监听**：自动监听Web所有页面（`/`、`/settings`）和H5页面（`/h5`）的用户访问，静默上报IP、页面路径、时间戳等信息，上报失败不影响正常使用
- **日访问量聚合**：后端按天聚合独立IP访问量，MongoDB新增`visit_logs`集合持久化存储。查询接口返回`{date, uv}`数组，支持指定日期范围（默认最近一年）
- **贡献墙展示**：使用ECharts heatmap实现7行（周一~周日）× N列（按周排列）的贡献格。每个小方块代表一天，颜色深浅表示当天独立访客数。顶部显示月份标签，左侧显示星期标签，底部图例注明活跃度等级
- **点击查看详情**：点击任意贡献格弹出浮层，显示该日期的IP访问详情列表（IP地址、访问页面、访问时间）

## 技术栈

- **后端**：NestJS 11 + TypeScript + Mongoose 9 + MongoDB
- **前端**：Vue 3.5 + Vite 8 + ECharts 6 + Tailwind CSS 4 + Axios 1.15

## 实现方案

### 后端：visit-log 模块（遵循现有 store-order 模式）

#### Schema设计

```typescript
@Schema({ collection: 'visit_logs', timestamps: true })
export class VisitLog {
  ip: string;            // 访问IP（必填，索引）
  page: string;          // 页面路径（必填，索引）
  userAgent?: string;    // User-Agent
  referer?: string;      // Referer
  timestamp: Date;       // 访问时间（必填，索引）
  date: string;          // YYYY-MM-DD 格式（必填，索引，便于按天聚合）
}
```

索引策略：`{date: 1}`, `{date: 1, ip: 1}` 复合索引加速聚合查询。

#### 接口设计

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/visit-log/report` | 上报访问，body:`{page}`，IP从请求头提取（优先`X-Forwarded-For`） |
| GET | `/api/visit-log/daily-stats?start=2025-06-01&end=2026-05-21` | 按天聚合独立IP数，返回`[{date, uv}]` |
| GET | `/api/visit-log/ip-detail?date=2026-05-21` | 查询指定日期的IP详情列表 |


#### Service核心逻辑

- `reportVisit`：提取真实IP → 写入`visit_logs`（异步，不等待）。
- `getDailyStats`：MongoDB Aggregation → `$match`日期范围 → `$group`按date和ip去重 → 二次`$group`按date统计uv → `$sort`升序。
- `getIpDetail`：`find({date})` → 按时间降序排列 → 返回ip/page/timestamp。

### 前端实现

#### 访问上报

- **Web端**：在`main.js`中使用`router.afterEach((to) => reportVisit({page: to.path}))`全局上报
- **H5端**：在`H5OnlineRate.vue`的`onMounted`中调用`reportVisit({page: '/h5'})`
- 上报使用`catch(() => {})`静默失败，不阻塞页面

#### API层（`frontend/src/api/visitLog.js`）

```javascript
import axios from 'axios'
const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000/api',
  timeout: 10000,
})
export const reportVisit = (data) => api.post('/visit-log/report', data)
export const getDailyStats = (start, end) => api.get('/visit-log/daily-stats', { params: { start, end } })
export const getIpDetail = (date) => api.get('/visit-log/ip-detail', { params: { date } })
```

#### 热力墙核心实现（StoreOrder.vue新增区块）

贡献墙采用GitHub风格布局：

- **Y轴（7行）**：周一~周日，与GitHub一致（从周日开始的calendar需要调整为周一优先）
- **X轴（N列）**：按周排列，每个col代表一周
- **数据转换**：将后台返回的`{date, uv}`数组转为heatmap格式`[[colIdx, rowIdx, uv]]`，其中`colIdx`=该日期所在周序号，`rowIdx`=星期几（0=周一）
- **缺失日期**：填充uv=0的格子（确保网格完整）
- **月份标签**：顶部定位标注每个月的起始位置

ECharts配置要点：

- `type: 'heatmap'`，配合`splitArea`实现网格分割线
- `visualMap`：color数组使用紫色到粉红色渐变`['#1e1b4b', '#6d28d9', '#a855f7', '#e879f9', '#f0abfc']`
- 图表背景色设为深色透明，由外层CSS控制深色容器
- tooltip显示日期 + 独立访客数
- 点击事件调用`getIpDetail`接口展示IP详情弹层

#### 深色主题样式

外层容器：`bg-gradient-to-br from-[#0f0a2e] to-[#1a1040] rounded-2xl shadow-xl`
内部标题区用白色半透明文字，图例区底部水平放置。

### 性能优化

- MongoDB索引覆盖date和ip字段，聚合查询走索引
- 上报接口异步写入，不等待DB返回即响应200
- ECharts使用`large: true`和`progressive: 200`优化渲染
- 日访问量数据默认缓存，页面切换时不重复请求
