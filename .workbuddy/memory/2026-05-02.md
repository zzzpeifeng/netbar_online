### Bug修复

#### 1. 日期选择不生效
- **问题**：选择日期后，表格及图表数据未更新
- **原因**：`watch(selectedDate, ...)` 没有正确监听 ref 的变化
- **修复**：改为 `watch(() => selectedDate.value, (newDate) => { ... })`
- **文件**：OnlineRateCompare.vue 第343-348行

#### 2. 门店筛选点击报错
- **问题**：点击门店筛选后报错 `Cannot read properties of undefined (reading 'length')`
- **原因**：Vue 模板中访问 ref 时会自动解包，不需要 `.value`，但代码错误地使用了 `tableData.value.length`
- **修复**：改为 `tableData.length`（共4处：表格筛选2处、图表筛选2处）
- **文件**：OnlineRateCompare.vue 第416、419、563、566行

## 经验教训
- Vue 3 模板中访问 ref 会自动解包，不要用 `.value`
- watch 监听 ref 时，需要使用函数返回值的方式：`watch(() => ref.value, callback)`
- 简约高级的UI设计要点：
  - 使用更大的圆角（rounded-xl, rounded-2xl）
  - 使用更精细的阴影（shadow-sm, shadow-lg）
  - 使用柔和的配色（slate 系代替 gray 系）
  - 使用更精细的 padding（p-3 代替 p-2）
  - 使用 transition-all 添加过渡效果

## 遇到的问题和解决方案

1. **MongoDB 认证失败**
   - 原因：密码中的 `@` 需要 URL 编码为 `%40%40`
   - 解决：使用 `netbar%40%40123456` 代替 `netbar@@123456`

2. **MongoDB 认证失败（续）**
   - 原因：用户 `admin` 是在 `admin` 数据库中创建的，需要指定 `authSource=admin`
   - 解决：在连接字符串中添加 `?authSource=admin`

3. **图表显示空白**
   - 原因：`updateChart()` 在 `chartInstance` 初始化之前被调用
   - 解决：使用 `watch(displayStores, () => { updateChart() }, { deep: true })` 响应式更新

4. **端口占用**
   - 原因：旧进程未清理
   - 解决：`pkill -f "nest start"` 和 `lsof -ti:3000 | xargs kill -9`
