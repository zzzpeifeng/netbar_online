### 1. 修复 MongoDB 连接问题
- **问题**：后端无法连接 MongoDB，报错 `Authentication failed`
- **原因**：`.env` 文件中的密码错误（密码是 `netbar@@123456`，不是 `netbar@123456`）
- **修复**：更新 `.env` 文件中的 `MONGODB_PASSWORD`
- **提交**：无（`.env` 文件在 `.gitignore` 中，不提交）

### 2. 修复 TypeScript 类型错误
- **问题**：构建时报错 `Argument of type 'string | number | boolean | undefined' is not assignable to parameter of type 'string | number | boolean'`
- **原因**：`config.get()` 返回类型可能是 `undefined`，而 `encodeURIComponent()` 不接受 `undefined`
- **修复**：将 URI 拼接拆分为多个步骤，每个配置项明确指定类型和默认值
- **提交**：`fix: 修复TypeScript类型错误，优化MongoDB连接配置`

### 3. 修复 CORS 跨域问题
- **问题**：前端请求后端 API 时报错 `Network Error`（CORS 策略阻止）
- **原因**：后端 `main.ts` 中 CORS 配置只允许 `localhost:5173`，但前端运行在 `5175` 端口
- **修复**：
  1. 第一次修改：改为 `origin: true`（允许所有来源）
  2. 第二次修改：明确指定可能的前端 URL（`5173`, `5174`, `5175`）
- **提交**：
  - `fix: 修复CORS配置（允许所有来源）`
  - `fix: 明确指定CORS允许的前端URL`

### 4. 修复图表初始化时序问题
- **问题**：前端控制台报错 `chartRef.value 不存在`，图表不显示
- **原因**：`onMounted` 中 `initChart()` 在 `fetchDates()` 之前执行，此时 `chartDisplayStores` 为空，导致图表 `div` 未渲染
- **修复**：修改 `onMounted`，先执行 `fetchDates()`，再用 `setTimeout` 延迟初始化图表
- **提交**：`fix: 修复CORS配置（允许所有来源），修复图表初始化时序问题`

## 部署相关

### 服务器部署步骤（记录）
1. **拉取最新代码**：`git pull origin main`
2. **构建前端**：
   ```bash
   cd /data/node_project/netbar_online/frontend
   npm install
   npm run build
   ```
3. **配置前端生产环境**：
   ```bash
   vim /data/node_project/netbar_online/frontend/.env.production
   # 内容：VITE_API_BASE_URL=http://8.138.164.134/api
   ```
4. **启动/重启后端**：
   ```bash
   cd /data/node_project/netbar_online/backend
   pm2 start ecosystem.config.js --env production
   pm2 save
   ```
5. **配置 Nginx**：
   - 复制 `nginx.conf.example` 到 `/etc/nginx/conf.d/netbar.conf`
   - 删除默认配置：`rm /etc/nginx/sites-enabled/default`
   - 测试配置：`sudo nginx -t`
   - 重启 Nginx：`sudo systemctl restart nginx`

## 技术笔记
- **CORS 配置**：NestJS 中使用 `app.enableCors({ origin: [...] })` 明确指定允许的源
- **Vue 3 图表初始化**：确保 DOM 已渲染后再初始化 ECharts，可以使用 `nextTick()` 或 `setTimeout()`
- **Tailwind CSS 响应式**：使用 `hidden`, `inline`, `sm:inline` 等类实现不同屏幕大小的显示/隐藏
- **MongoDB 连接**：使用 `@nestjs/config` 的 `ConfigModule`，在 `useFactory` 中拼接连接字符串
- **环境变量**：Vite 使用 `import.meta.env.VITE_*` 访问环境变量，需要创建 `.env.development` 和 `.env.production` 文件
